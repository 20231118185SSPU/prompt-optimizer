# ALIGN-DECISIONS.md：重大决策日志模板

> 本文件是 `.align/decisions.log.md` 的模板。由 `align-init` skill 生成初始文件，由沉淀门（门 5）自动追加。
> 大小纪律：每条 ≤5 行，超 100 条归档。

## 用途

记录影响后续方向、架构、权限或高风险边界的重大决策，让 agent 理解"为什么这样决定"。档位 C 澄清的结论自动归档到本文件。

## 由谁生成/更新

- **生成**：`align-init` skill 创建初始空文件。
- **更新**：沉淀门（门 5）在出现高风险决策确认或档位 C 澄清结论时自动追加。
- **不写入的内容**：日常执行决策（放 lessons.md）、空洞总结、未确认的推断、一次性任务信息。

## 填写标准

每条必须包含四个要素：决策内容、影响、依据、日期。格式固定为：

```text
- [决策类型] 决策内容：xxx
  影响：xxx
  依据：xxx（用户确认/踩坑验证/新约定）
  日期：yyyy-mm-dd
```

判定标准：agent 读到后能理解这个决策为什么做、影响什么、下次遇到类似场景该怎么处理。

---

## 写入格式

### 标准格式（一条 ≤5 行）

```text
- [架构决策] 认证状态存储改为 httpOnly cookie
  影响：所有 auth 相关任务，不再使用 localStorage 存登录状态
  依据：用户确认（档位 C 澄清，安全风险）
  日期：2026-07-05

- [数据决策] 生产数据库使用 MySQL 8.0 非 PostgreSQL
  影响：SQL 语法按 MySQL 兼容写，不使用 PostgreSQL 特有函数
  依据：踩坑验证（假设使用 PostgreSQL 被证伪）
  日期：2026-07-05

- [权限决策] 禁止 agent 直接执行 DROP 语句
  影响：涉及删除表/列时必须先确认备份
  依据：新约定（安全规范）
  日期：2026-07-05
```

### 不合格格式

```text
- 决定使用 React  ← 缺少影响、依据、日期
- 今天和团队讨论了架构  ← 空洞记录，无决策内容
- 用户说这个功能要做  ← 不是决策，是需求
```

---

## 膨胀控制

- 每条不超过 5 行。
- 超 100 条时，将最旧 40 条移到 `.align/archive/decisions-{yyyy-mm-dd}.md`。
- 归档文件不再被 agent 自动读取。
- 归档操作在门 5 沉淀门执行时自动完成。

---

## 沉淀触发条件

本文件在以下情况追加：

1. **高风险决策确认**：档位 C 澄清的结论涉及高风险操作（生产/数据库/删除/不可逆），用户已确认方向。
2. **架构决策**：影响后续开发方向的技术选型或架构变更。
3. **权限决策**：影响 agent 执行权限边界的约定。
4. **推翻假设**：原有 `[假设]` 被证伪，且影响项目方向。

详细触发判定标准见 `core/protocol/07-precipitation.md`。

## 与 lessons.md 的区别

| 属性 | lessons.md | decisions.log.md |
| --- | --- | --- |
| 内容 | 执行规则（"下次怎么做"） | 决策记录（"为什么这样定"） |
| 长度 | 每条 ≤2 行 | 每条 ≤5 行 |
| 读取顺序 | 第一位（最优先） | 不在自动读取顺序中（按需读取） |
| 触发 | 踩坑/纠正/新约定/推翻假设 | 高风险决策确认/架构决策/权限决策 |
| 归档 | 50 条归档 | 100 条归档 |

## 模板使用说明

`align-init` 生成 `.align/decisions.log.md` 时，从本模板复制结构，初始内容为空（只有标题和说明，无决策条目）。

沉淀门追加时，在"决策记录"区域末尾添加新条目，不修改已有条目（除非需要纠正错误决策）。

纠正错误决策时，在该条目后追加纠正记录：

```text
- [纠正] 上述 [架构决策] 认证状态存储改为 httpOnly cookie 已撤销
  原因：发现 SSR 场景需要 localStorage 降级
  日期：yyyy-mm-dd
```
