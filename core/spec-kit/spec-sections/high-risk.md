# 高风险操作清单预设

> 规范章节库：高风险操作清单。每个预设提供可验证的规范条目。
> 高风险清单是三档路由安全阀的判断依据：命中时必须停下问用户（档位 C）。

## 预设 A：通用高风险（推荐，适用于所有项目）

```text
- 数据库 schema 变更：必须先在 staging 验证，必须有回滚脚本
- 生产环境部署：必须先确认版本号和变更清单
- 删除文件/目录：必须先确认是否有引用，必须 git commit 后再删
- 修改 public API：必须同步更新类型定义和调用方
- 引入新依赖：必须确认包体积、维护状态和许可证
```

判定标准：agent 在执行这些操作前必须停下问用户（档位 C），不得静默执行。

## 预设 B：认证/支付项目（在 A 基础上追加）

```text
（继承预设 A 全部条目，追加以下）
- 修改认证/权限逻辑：必须跑完整 auth 测试 + 人工评审
- 修改支付流程：必须在沙箱环境验证，必须人工评审
- 修改用户数据访问：必须确认权限范围，必须人工评审
- 修改加密/密钥相关：必须确认安全影响，必须人工评审
```

判定标准：agent 在执行这些操作前必须停下问用户（档位 C），不得静默执行。

## 预设 C：基础设施项目（在 A 基础上追加）

```text
（继承预设 A 全部条目，追加以下）
- 修改 CI/CD 配置：必须先在分支验证，不得直接改 main 上的配置
- 修改 Dockerfile：必须先本地构建验证
- 修改基础设施代码（Terraform/Pulumi）：必须先 plan 验证
- 修改环境变量配置：必须确认不影响生产环境
- 执行数据库迁移脚本：必须先备份，必须有回滚脚本
```

判定标准：agent 在执行这些操作前必须停下问用户（档位 C），不得静默执行。
