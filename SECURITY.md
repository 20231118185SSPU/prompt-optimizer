# 安全策略

## 支持范围

安全修复以当前默认分支为准。仓库尚未发布固定 release 或 tag，因此旧快照不承诺回补；发布后必须在此列出受支持版本。

## 报告漏洞

请通过 GitHub 仓库的私密安全报告功能提交漏洞，禁止在公开 issue 中粘贴 token、密钥、用户数据、生产日志或可直接利用的细节。

报告至少包含：

- 受影响版本、commit 或安装方式。
- 受影响宿主与能力等级。
- 最小复现步骤和实际影响。
- 已完成的脱敏处理。
- 可行的缓解或回滚方式（如已知）。

在维护者确认修复或披露窗口前，请勿公开利用细节。若仓库未启用私密安全报告，先创建不含敏感信息的 issue，只说明需要安全联系方式。

## 敏感信息规则

- `.align/` 只允许提交无敏感摘要。
- completion evidence 默认本地保存，仓库中只能保存脱敏摘要或外部引用。
- 评测 runner 必须脱敏 API key、Bearer token 和认证错误输出。
- 生产变更、数据删除、发布和 push 必须经过显式授权，`[直出]` 不得绕过。
