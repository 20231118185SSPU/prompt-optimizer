# 意图偏差案例库

> 10 个"用户说 X 其实要 Y"的案例，用于识别意图-表达偏差，并把真实目标、约束和验收写入 Agent Brief。

---

## 案例 01：正则解析 HTML

**用户原话**：帮我用正则解析这个 HTML，提取商品价格。

**偏差类型**：解决方案伪装成需求（XY Problem）

**可能真实意图**：稳定提取商品价格；正则只是用户预设的实现方式，可能不适合复杂 HTML。

**推荐处理**：输出 A/B 确认路径：A. 按用户要求写正则；B. 改用 HTML parser / DOM 解析方案。

**应写入 Agent Brief 的约束**：不得把"提取网页数据"当作已确认事实；如用户选择 B，需说明依赖、输入样例和失败场景。

---

## 案例 02：全局变量改单例

**用户原话**：帮我把这个全局变量改成单例。

**偏差类型**：解决方案伪装成需求（XY Problem）

**可能真实意图**：解决状态管理混乱、初始化顺序不稳定或跨模块共享状态不可控。

**推荐处理**：先确认目标是封装访问、限制实例数量，还是重构状态管理边界。

**应写入 Agent Brief 的约束**：保留现有 public API 或明确列出调用方迁移；新增单例前先说明替代方案和风险。

---

## 案例 03：缓存所有接口

**用户原话**：帮我给所有接口加缓存，页面太慢了。

**偏差类型**：解决方案伪装成需求（XY Problem）

**可能真实意图**：降低页面等待时间；性能瓶颈可能来自慢查询、重复请求、前端渲染或网络延迟。

**推荐处理**：先做性能定位，再决定缓存、查询优化、请求合并或前端优化。

**应写入 Agent Brief 的约束**：缓存方案必须说明失效策略、数据一致性风险和验证指标。

---

## 案例 04：加 loading 动画

**用户原话**：这个页面加载慢，帮我加个 loading 动画。

**偏差类型**：症状伪装成原因

**可能真实意图**：降低用户等待焦虑，或真正解决接口 / 渲染性能问题。

**推荐处理**：在背景中记录"页面加载慢"和"可能根因是接口性能或渲染阻塞"，要求 agent 先诊断根因再决定修复层次。

**应写入 Agent Brief 的约束**：允许加 loading 只能作为体验补偿；不得用动画掩盖可修复的性能问题。

---

## 案例 05：隐藏报错信息

**用户原话**：帮我把这个报错信息隐藏掉，用户看着烦。

**偏差类型**：症状伪装成原因

**可能真实意图**：改善用户体验；真实问题可能是错误本身、错误频率过高或错误文案不适合用户。

**推荐处理**：先定位错误来源和影响范围，再选择修复错误、降级提示或用户友好的错误展示。

**应写入 Agent Brief 的约束**：不得吞掉异常；日志、监控和用户可理解提示必须保留。

---

## 案例 06：重试解决支付失败

**用户原话**：支付偶尔失败，帮我加三次重试。

**偏差类型**：症状伪装成原因

**可能真实意图**：提高支付成功率；失败可能来自幂等缺失、第三方限流、网络超时或参数错误。

**推荐处理**：先区分可重试和不可重试错误，再决定重试策略。

**应写入 Agent Brief 的约束**：支付相关改动必须先确认幂等键、重复扣款防护、日志和回滚条件。

---

## 案例 07：函数参数改对象

**用户原话**：把这个函数的参数改成对象，调用起来更优雅。

**偏差类型**：局部视角遮蔽全局目标

**可能真实意图**：提升可读性和可扩展性；但参数签名变化会影响所有调用方和测试。

**推荐处理**：先列出所有调用点和兼容策略，再决定是否分阶段迁移。

**应写入 Agent Brief 的约束**：必须做影响面分析；超出当前范围的调用方变更需要先报告。

---

## 案例 08：配置改环境变量

**用户原话**：把这个配置改成环境变量。

**偏差类型**：局部视角遮蔽全局目标

**可能真实意图**：支持多环境部署或避免硬编码；但会影响本地开发、CI、部署文档和密钥管理。

**推荐处理**：先确认配置类型、默认值、安全级别和部署链路，再修改读取方式。

**应写入 Agent Brief 的约束**：必须更新示例环境变量、文档、测试和部署检查；不得把敏感值写入仓库。

---

## 案例 09：数据库字段改名

**用户原话**：把 user 表里的 name 字段改成 display_name。

**偏差类型**：局部视角遮蔽全局目标

**可能真实意图**：统一命名或区分真实姓名与展示名；字段改名会影响迁移、接口、前端、报表和历史数据。

**推荐处理**：先做影响面分析，确认是否需要兼容期、数据迁移和 API 版本策略。

**应写入 Agent Brief 的约束**：涉及数据库迁移时必须说明回滚方案、兼容策略和验证 SQL / 测试。

---

## 案例 10：全面优化项目

**用户原话**：帮我全面优化这个项目，让它更好。

**偏差类型**：混合偏差：高抽象动词 + 缺少 WHY + 局部/全局边界不明

**可能真实意图**：可能是性能、可维护性、用户体验、代码质量或交付速度中的一种或多种。

**推荐处理**：进入意图决策树，只问一个最高价值问题：这次优化第一优先级是什么？

**应写入 Agent Brief 的约束**：未确认第一优先级前不得输出最终执行方案；确认后锁定范围、交付物和验收标准。
