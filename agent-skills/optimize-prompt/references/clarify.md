# 澄清访谈模板

> 用于用户还没想清楚，或需求缺口会导致 agent 执行偏差的场景。

## 触发条件

当以下信息缺失且会改变结果时，先澄清，不要直接执行：

- 目标到底是什么
- 最终交付物是什么
- 是否允许修改文件、联网、安装依赖、提交代码
- 任务范围和排除项
- 成功标准
- 数据、安全、权限、成本相关约束

## 访谈规则

1. 一次只问一个问题。
2. 问最高价值的问题。
3. 每个问题都必须说明为什么重要。
4. 每个问题都提供一个推荐答案。
5. 能通过读文件、读文档、查代码获得的信息，不问用户。
6. 用户回答后，更新理解，再问下一个关键问题或输出最终 Agent Brief。

## 输出格式

```markdown
## 当前理解

[用 2-4 句话复述你认为用户想做什么]

## 最大不确定点

[只写一个最影响结果的不确定点]

## 我的问题

[只问一个问题]

## 推荐答案

[给出默认建议，并说明理由]
```

## 问题优先级

1. 目标问题：你最终想让用户/系统/项目发生什么变化？
2. 交付物问题：你希望 agent 最后交付什么？
3. 范围问题：哪些内容必须做，哪些不能碰？
4. 验收问题：什么结果算完成？
5. 风格问题：输出应该偏简洁、详细、正式还是执行导向？
